# Engines of Experience (Tynan Sylvester)

From "Designing Games: A Guide to Engineering Experiences"

## Core Thesis

**Games are artificial systems for generating experiences.** Unlike other media that author events directly, games design mechanics that generate events during play.

## Mechanics → Events → Emotions → Experience

```
Designer creates MECHANICS (rules)
    ↓
Mechanics + Players generate EVENTS
    ↓
Events trigger EMOTIONS (via human value changes)
    ↓
Emotions merge into integrated EXPERIENCE
```

## The Primacy of Emotion

- Events must provoke **emotion** to be meaningful
- Most game emotions are subtle, below conscious awareness
- "Fun" is just one emotion—games can provoke triumph, terror, contemplation, grief
- **Designer skill:** Detecting subtle emotions (a flicker of anger, pulse of triumph)

## Human Values & Emotional Triggers

**Human Value:** Anything important to people that can shift between states.

| Human Value | States |
|-------------|--------|
| life/death | alive ↔ dead |
| victory/defeat | winning ↔ losing |
| friend/enemy | bonded ↔ hostile |
| wealth/poverty | rich ↔ poor |
| safety/danger | secure ↔ threatened |
| knowledge/ignorance | informed ↔ clueless |

**Key insight:** The more important the human value and the more it changes, the greater the emotion.

### Anticipation Creates Emotion Too

Emotions appear not just in response to change, but in **anticipation** of change. This creates suspense—sensing you're on the knife edge between two drastically different outcomes.

### Reveals Equal Changes

Information reveals are emotionally equivalent to actual changes. Learning you lost $1000 feels the same as losing $1000 in the moment.

## Basic Emotional Triggers

| Trigger | How It Works |
|---------|--------------|
| **Learning** | Epiphany, insight, skill mastery |
| **Character Arcs** | Empathy with characters' struggles |
| **Challenge** | Focus, triumph, wanting to try again |
| **Social Interaction** | Status, trust, competition, cooperation |
| **Acquisition** | Getting richer (even virtually) |
| **Music** | Continuous emotional coloring |
| **Spectacle** | Explosions, big moments (but shallow) |
| **Beauty** | Pleasure from perception |
| **Environment** | Savanna comfort, tundra isolation |
| **Primal Threats** | Spiders, gore, disease (easily overused) |

## The Fiction Layer

**Fiction wraps mechanics** to add a second layer of emotional meaning.

- Mario isn't a plumber—he's a collision cylinder
- The Sims teenager isn't falling in love—the game flipped some bits
- But the make-believe still creates emotional echoes of real feelings

**Fiction vs Mechanics tradeoffs:**
- Mechanics can do: tension, triumph, learning, social interaction
- Fiction can do: character, plot, humor, awe, immersion
- They can interfere: game clichés (crates, amnesia, super-soldiers) exist because they justify good mechanics

**Pinnacle of design:** Threading fiction and mechanics into a single system of meaning.

## Constructing Experiences

### Techniques

| Technique | Description |
|-----------|-------------|
| **Pure Emotion** | Stack multiple triggers for same feeling |
| **Juxtaposition** | Combine incompatible feelings (Gears of War: mourning + violence) |
| **Antagonistic Emotions** | Some emotions cancel out (social fun vs ruthless competition) |
| **Atmosphere** | Spread-out emotional background |
| **Emotional Variation** | Classic pacing curve: hook → rising action → climax → denouement |

### Flow

**Flow:** State of concentration so focused it amounts to total absorption.

- Time disappears
- Real world fades away
- Built on continuous stream of tiny successes

**Flow appears when:** Challenge is perfectly balanced against ability level.
- Too hard → anxiety, confusion
- Too easy → boredom

### Immersion

**Immersion:** When player's experience mirrors the character's experience.

**Two-Factor Theory of Emotion:**
1. Flow strips away real world, creates arousal state
2. Mechanics generate raw physiological arousal
3. Fiction layer labels that arousal (as fear, determination, etc.)

Result: Player sees, hears, and feels what character does. Player *is* the character.

## Key Mantras

- "Game designers don't design events. We design systems that generate events."
- "What really matters is how a game makes us feel."
- "The emotions of play are not limited to 'fun.'"
- "Fiction and mechanics need not fight—they can enhance each other."
