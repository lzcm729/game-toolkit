# 提取 Prompt 模板

启动 Explore agent 时使用此模板。将 `{DOC_LIST}` 和 `{FOCUS_AREAS}` 替换为实际内容。

---

你是设计文档结构化提取器。逐一阅读以下文档，提取所有可交叉比对的定义。

**文档清单：**
{DOC_LIST}

**请逐一阅读每个文件，然后按以下格式提取：**

## 1. 数值定义
列出所有具体数字（金额、百分比、天数、次数、阈值等），格式：
- `[来源文件名] 概念: 数值`

## 2. 概念定义
列出所有关键概念的定义，格式：
- `[来源文件名] 概念名: 定义内容`

重点关注：{FOCUS_AREAS}

## 3. 流程与规则
列出状态转换、触发条件、因果关系，格式：
- `[来源文件名] 规则: 描述`

## 4. 枚举列表
列出所有分类、类型、档位等列表，格式：
- `[来源文件名] 列表名: [项1, 项2, ...]`

## 5. 组内矛盾
如果同组文档之间有不一致的地方，列出：
- 文档A说: ...
- 文档B说: ...
- 矛盾点: ...

## 6. 模糊/未定义
定义不清、有歧义、缺少具体数值的地方：
- `[来源文件名] 问题: 描述`

**重要：只提取，不评价。尽量完整，不要遗漏任何数值或定义。**
