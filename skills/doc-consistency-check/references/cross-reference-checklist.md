# 交叉比对检查清单

阶段2汇总时，逐项检查以下维度。

## 数值一致性

- [ ] 同一操作（如"高利贷交易"）在不同文档中的数值是否相同
- [ ] 同一参数（如"初始资金"）只有一个权威来源，还是多处定义
- [ ] 百分比/概率的基数是否一致（如"赎回率60%"的基数是什么）
- [ ] 成本/收益数值是否在合理范围内（如总投资额 vs 游戏目标金额）

## 术语一致性

- [ ] 同一概念是否在所有文档中使用相同名称
- [ ] 是否有改名后的残留（旧名称仍在某些文档中出现）
- [ ] 计数方向是否一致（如"高=好"还是"高=坏"）
- [ ] 英文术语和中文术语是否一一对应

## 枚举对齐

- [ ] 分类列表的成员数是否一致（如"合同档位"各处都是4档？）
- [ ] 枚举成员的名称是否一致（如 CHARITY vs charity vs 慈善）
- [ ] 是否有某文档引用了其他文档已删除的枚举成员

## 流程一致性

- [ ] 同一流程在不同文档中的步骤顺序是否相同
- [ ] 触发条件是否互相矛盾（如文档A说"阈值触发"，文档B说"日期触发"）
- [ ] 状态转换规则是否有冲突

## 跨系统接口

- [ ] 系统A输出的变量名与系统B消费的变量名是否匹配
- [ ] 系统间传递的数据格式是否定义清晰
- [ ] 是否有系统依赖了另一个系统未定义的功能

## 版本同步

- [ ] 是否有文档引用了已过时的格式（如旧版枚举格式）
- [ ] 修订历史中的变更是否已同步到所有相关文档
- [ ] 样例文档是否与系统文档使用的术语一致

## 缺失检测

- [ ] 核心参数是否有统一的权威来源（如经济参数应该是数值权威）
- [ ] 是否有参数只在一处定义但被多处引用
- [ ] 是否有文档引用了不存在的概念或变量
