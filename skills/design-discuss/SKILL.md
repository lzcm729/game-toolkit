# Design Discuss

游戏设计讨论与点子收录工具。

## 模式判断

根据用户意图选择模式：

| 用户意图 | 模式 | 加载内容 |
|----------|------|----------|
| 讨论、评估、理论咨询 | 讨论模式 | 先查 `docs/IDEAS.md`，再读 `references/theory-framework.md` |
| 记录、收录、记下来 | 记录模式 | 读取 `references/idea-format.md` |
| 讨论后想记录结论 | 混合 | 已有理论框架，直接记录 |

---

## 讨论模式

**触发词：** 讨论、设计、评估、合理吗、好玩吗、怎么设计、哪个好

**流程：**
1. 检查 `docs/IDEAS.md` 是否存在相关点子（用关键词搜索）
   - 如有相关记录 → 告知用户并询问是否基于已有点子讨论
   - 如无相关记录 → 继续
2. 读取 `references/theory-framework.md` 获取完整设计理论框架
3. 理解用户的设计需求
4. 应用理论框架分析
5. 提出建议和方案
6. 讨论结束时输出设计结论

**讨论结束输出：**
```
## 设计结论

**核心决策：** [一句话描述]

**关键要点：**
- [要点1]
- [要点2]

**实现建议：**
- framework: [需要实现的逻辑]
- content: [需要填充的内容]
- interaction: [需要创建的 UI]
```

---

## 记录模式

**触发词：** 记录、收录、记下来、存一下、加到点子

**流程：**
1. 读取 `references/idea-format.md` 获取格式规范
2. 从用户输入提取：标题、描述、相关系统、设计意图
3. 检查与现有设计的冲突（可选）
4. 追加到 `docs/IDEAS.md`

**快速记录（无冲突检测）：** 用户明确说"直接记录"时跳过冲突检测。

---

## 混合场景

讨论过程中产生值得记录的点子：
- 询问用户："这个想法要记录到 IDEAS.md 吗？"
- 用户同意 → 直接追加，无需再读取 idea-format.md
