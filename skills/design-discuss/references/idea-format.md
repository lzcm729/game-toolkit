# 点子记录格式

## 文件位置

`docs/IDEAS.md`

## 记录格式

```markdown
## [点子标题]

**日期：** YYYY-MM-DD
**相关系统：** [系统名称]
**状态：** 待评估 | 待讨论 | 已采纳 | 已放弃

### 描述

[详细描述]

### 设计意图

[这个点子想解决什么问题/带来什么体验]

### 备注

[冲突信息、讨论记录等]

---
```

## 首次创建文件

如果 `docs/IDEAS.md` 不存在，先创建头部：

```markdown
# 设计点子收录

本文档收录开发过程中产生的游戏设计点子，供后续评估和整合。

**状态说明：**
- 待评估：新记录的点子，尚未深入讨论
- 待讨论：与现有设计有冲突，需要进一步讨论
- 已采纳：决定实现，已整合到设计文档
- 已放弃：经讨论后决定不采用

---

```

## 冲突检测

记录前检查设计文档目录：
`C:\Users\lzcm7\OneDrive\GameProject\Godot\PawnShop_Claude\Designer\系统设计文档\`

如果发现冲突，使用 AskUserQuestion 询问用户处理方式：
1. 仍然记录（标记为"待讨论"）
2. 放弃这个点子

## 原则

1. **不丢失任何想法** — 即使有冲突，也先记录
2. **标记冲突状态** — 明确标注需要讨论的点子
3. **保持简洁** — 抓住核心即可
