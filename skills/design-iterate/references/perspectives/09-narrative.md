# 叙事整合与玩家能动性 (Narrative)

**理论基础**: Rouse 叙事理论（设计师故事/玩家故事）、Schell Lens #73 故事机器、Lens #78 故事、Lens #79 自由、Lens #81 间接控制、Lens #82 共谋

**关注点**:
- 设计师预设叙事与玩家自生叙事的融合度
- "展示而非讲述"：文本/对话是否可以用环境/行为/后果替代
- 每个NPC角色是否有清晰的功能和内在矛盾
- 系统是否能像"故事机器"一样自动生成有趣的叙事
- 玩家是否感觉自己在"创造"故事而非"消费"故事
- 叙事机制是否打断了玩法（非交互式叙事的时长和频率）

**Prompt 模板**:
```
从【叙事整合与玩家能动性】视角评审此设计文档（基于 Rouse + Schell 叙事理论）。重点关注：
- 设计师故事 vs 玩家故事：预设剧情和玩家自生叙事如何交织？
- 展示 vs 讲述：哪些信息通过文本传达？能否通过环境/行为/后果替代？
- 角色设计：每个NPC是否有清晰功能（推动系统/提供信息/制造紧张/提供情感）？
- 角色矛盾：NPC是否有内在矛盾让他们感觉真实（不是单维度的"好人/坏人"）？
- 故事机器：系统规则是否能自动产生有叙事价值的涌现故事？
- 玩家能动性：玩家是否感觉叙事结果源于自己的选择而非脚本？
- 非交互叙事开销：文本/对话/闪现等非交互时刻是否打断了游戏心流？
提出具体的叙事整合问题和改进建议，标注严重程度（🔴高/🟡中/🟢低）。
```
