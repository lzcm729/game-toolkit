# 认知负荷与可学习性 (Cognitive)

**理论基础**: Schell Lens #48 简单/复杂、Lens #62 透明性、Lens #66 通道与维度、Lens #67 模式；GameFlow 模型

**关注点**:
- 玩家在任何时刻需要同时记住多少条规则
- 信息是否按优先级分层展示
- 新机制引入时是否有渐进披露
- 界面模式是否过多或容易混淆
- 系统的"心智模型"是否直觉可构建
- 规则例外的数量

**Prompt 模板**:
```
从【认知负荷与可学习性】视角评审此设计文档（基于 Schell 界面理论 + GameFlow 模型）。重点关注：
- 规则总量：玩家需要记住多少条独立规则才能有效使用这个系统？
- 渐进披露：系统是否支持分阶段学习？还是必须一次性理解全貌？
- 心智模型：玩家能否用一句简单的话概括系统运作方式？
- 信息层级：最关键的信息是否在最显眼的位置？次要信息是否适当隐藏？
- 例外数量：基础规则有多少特殊情况/例外？每个例外增加多少认知负担？
- 模式清晰度：不同操作模式之间的区别是否显而易见？
- 反馈可读性：玩家的行为结果是否立即可理解？还是需要"查攻略"才知道发生了什么？
- 首次体验：一个完全不看说明的新玩家会如何理解这个系统？
提出具体的认知负荷问题和简化建议，标注严重程度（🔴高/🟡中/🟢低）。
```
